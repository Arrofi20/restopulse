# RestoPulse Deployment

## Render Services

| Service | Type | Expected URL |
|---------|------|--------------|
| restopulse-api | Web Service (Node.js) | https://restopulse-api.onrender.com |
| restopulse-web | Static Site | https://restopulse-web.onrender.com |
| restopulse-db | PostgreSQL | (internal) |

## Verification Commands

After deploying via the Render dashboard (Blueprint from `render.yaml`), run the following checks:

### Backend Health Check

```bash
curl -s https://restopulse-api.onrender.com/health
```

**Expected:** HTTP 200 with JSON body containing `status: 'ok'`.

Example expected response:
```json
{"status":"ok","timestamp":"2026-06-27T12:00:00.000Z"}
```

### Frontend Availability

```bash
curl -s https://restopulse-web.onrender.com/ | grep -o 'RestoPulse'
```

**Expected:** HTTP 200 with HTML containing the text `RestoPulse`.

## Deployment Steps

1. Push this repository to GitHub.
2. In the Render dashboard, create a new Blueprint from `render.yaml`.
3. Wait for the `restopulse-api` Web Service and `restopulse-db` PostgreSQL database to finish building.
4. Wait for the `restopulse-web` Static Site to finish building.
5. Run the verification commands above.
6. Update this file with the actual assigned URLs if they differ from the expected ones above.

## Environment Variables

The following env vars are configured in `render.yaml` and will be auto-populated by Render:

- `JWT_SECRET` — auto-generated by Render
- `CORS_ORIGIN` — set to `https://restopulse-web.onrender.com`
- `VITE_API_BASE_URL` — set to `https://restopulse-api.onrender.com/api`
- `PORT` — set to `10000`
- `DB_PROVIDER` — set to `postgresql`
