# RestoPulse Deployment

## Railway Services

| Service | Type | Expected URL |
|---------|------|--------------|
| restopulse-api | Web Service (Node.js) | https://restopulse-api.up.railway.app |
| restopulse-web | Static Site | https://restopulse-web.up.railway.app |
| restopulse-db | PostgreSQL | (internal) |

## Verification Commands

After deploying via the Railway dashboard, run the following checks:

### Backend Health Check

```bash
curl -s https://restopulse-api.up.railway.app/health
```

**Expected:** HTTP 200 with JSON body containing `status: 'ok'`.

Example expected response:
```json
{"status":"ok","timestamp":"2026-06-27T12:00:00.000Z"}
```

### Frontend Availability

```bash
curl -s https://restopulse-web.up.railway.app/ | grep -o 'RestoPulse'
```

**Expected:** HTTP 200 with HTML containing the text `RestoPulse`.

## Deployment Steps

1. Push this repository to GitHub.
2. In the Railway dashboard, create a new project and add the repository.
3. Add a PostgreSQL database service named `restopulse-db`.
4. Configure the required environment variables in the Railway dashboard for the `restopulse-api` service.
5. Wait for the `restopulse-api` Web Service and `restopulse-db` PostgreSQL database to finish building.
6. Deploy the `restopulse-web` Static Site and wait for it to finish building.
7. Run the verification commands above.
8. Update this file with the actual assigned URLs if they differ from the expected ones above.

## Environment Variables

The following env vars should be configured in the Railway dashboard for the `restopulse-api` service:

- `JWT_SECRET` — auto-generated by Railway or set manually
- `CORS_ORIGIN` — set to `https://restopulse-web.up.railway.app`
- `VITE_API_BASE_URL` — set to `https://restopulse-api.up.railway.app/api`
- `PORT` — set to `3000` (or leave default)
- `DB_PROVIDER` — set to `postgresql`
