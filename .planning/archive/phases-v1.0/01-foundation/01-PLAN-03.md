- Import { Decimal } from '@prisma/client/runtime/library' (or appropriate path for Prisma 7.8 Decimal)
- Import DailySalesRepository, StatusLogRepository, SalesTrendRepository from '../repositories'
- Import { createSalesSchema } from '../validation/sales.schema'
- Class SalesService with constructor accepting repositories
- Method createSale(data: { date: string; revenue: number; top_menu_items: string[]; outlet_id: string; actor_id: string; data_source?: string }):
  1. Validate input with createSalesSchema.parse(data)
  2. Convert date string to Date object: const saleDate = new Date(data.date + 'T00:00:00Z')
  3. Business rule: if saleDate > new Date(), throw Error("Cannot record sales for future dates")
  4. Check for existing sale on same date: dailySalesRepo.findByOutletAndDate(data.outlet_id, saleDate)
  5. If exists, throw Error("Sales record already exists for this date")
  6. Create sale: dailySalesRepo.create({ date: saleDate, revenue: new Decimal(data.revenue), top_menu_items: data.top_menu_items, outlet_id: data.outlet_id, data_source: data.data_source || 'REAL' })
  7. Create audit log: statusLogRepo.create({ action: 'CREATE', entity_type: 'DailySales', entity_id: createdSale.id, new_value: createdSale, actor_id: data.actor_id })
  8. Update trend: salesTrendRepo.upsert({ date: saleDate, revenue: new Decimal(data.revenue), menu_popularity: computeMenuPopularity(data.top_menu_items), outlet_id: data.outlet_id })
  9. Return createdSale

- Method getSalesByRange(outlet_id: string, start: string, end: string):
  1. Validate with dateRangeSchema.parse({ start, end })
  2. Convert to Date objects
  3. Return dailySalesRepo.findByDateRange(outlet_id, startDate, endDate)

- Helper function computeMenuPopularity(menuItems: string[]): object:
  - Count occurrences of each menu item
  - Return object with items array: [{ name: string, count: number, percentage: number }]
  - Percentage = (count / total items) * 100, rounded to 2 decimals

Note: Use new Decimal(data.revenue) for Prisma Decimal fields. If Prisma 7.8 uses a different Decimal import path, check node_modules/@prisma/client/runtime/ for the correct export.
