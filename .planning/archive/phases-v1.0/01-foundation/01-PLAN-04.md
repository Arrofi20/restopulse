Implement the dummy data injector that generates realistic historical sales data for demo and onboarding purposes. The injector must be admin-gated, require typed confirmation, and never pollute real data.

Purpose: Restaurant owners are time-poor and will abandon the product if manual entry feels burdensome. The dummy injector seeds the first week (or full year) so the dashboard is meaningful from day one. It also enables demo presentations without waiting for real data accumulation.
Output: DummyService, DummyController, and protected admin route for dummy injection.
</objective>

- Method injectDummyData(outlet_id: string, days: number, confirm: string): Promise&lt;{ inserted: number }&gt;:
  1. Verify confirm === "HAPUS"; if not, throw Error("Confirmation required: type 'HAPUS' to proceed")
  2. Cap days at 730 (2 years max) to prevent abuse
  3. Calculate startDate = today - days
  4. For each day from startDate to today:
     a. Generate day data via generateDayData
     b. Check if a DUMMY record exists for this date: dailySalesRepo.findByOutletAndDate(outlet_id, currentDate)
     c. If exists and data_source === 'DUMMY', update it with new generated data
     d. If exists and data_source === 'REAL', SKIP (never overwrite real data)
     e. If not exists, create new DUMMY record
     f. Upsert SalesTrend for this date
  5. Return { inserted: count of created/updated records }

- Method clearDummyData(outlet_id: string): Promise&lt;{ deleted: number }&gt;:
  - Find all DailySales where outlet_id and data_source === 'DUMMY'
  - Delete them and corresponding SalesTrend records
  - Return { deleted: count }
  - Note: This is a safety method; may not be exposed via API in v1
