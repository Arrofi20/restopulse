## v1 Requirements (Complete)

### Authentication

- [x] **AUTH-01**: User dapat masuk ke sistem menggunakan akun (username & password) yang terdaftar — *Completed in Phase 1*
- [x] **AUTH-02**: Sistem memvalidasi hak akses login dan mengarahkan pemilik ke halaman Dasbor Utama — *Completed in Phase 1*

### Dashboard Analytics

- [x] **DASH-01**: Sistem mengagregasikan data omset dan menggambar Line Chart interaktif riwayat tren harian secara otomatis saat halaman dasbor dimuat — *Completed in Phase 2*
- [x] **DASH-02**: Sistem mengagregasikan data performa menu dan menggambar Pie Chart persentase menu terlaris secara otomatis — *Completed in Phase 2*
- [x] **DASH-03**: Sistem menampilkan pop-up detail (tooltip) berisi nominal angka omset dan menu terlaris secara instan ketika titik tanggal pada grafik disentuh — *Completed in Phase 2*

### Data Entry & Simulation

- [x] **DATA-01**: Pemilik/pengembang dapat menyuntikkan ratusan baris data transaksi fiktif harian masa lalu secara berurutan ke database melalui tombol "Suntik Data Simulasi" — *Completed in Phase 1*
- [x] **DATA-02**: Pemilik dapat memasukkan data transaksi harian baru secara manual melalui formulir (Tanggal, Omset, Menu Terlaris) — *Completed in Phase 1*

### E-Report & Export

- [x] **REPT-01**: Sistem menyaring, menghitung total pendapatan, dan menyusun lembar ringkasan laporan digital di layar sesuai filter rentang tanggal pemilik — *Completed in Phase 3*
- [x] **REPT-02**: Sistem mengonversi struktur data laporan di layar menjadi file dokumen digital siap cetak berformat PDF — *Completed in Phase 3*

## v2 Requirements (Deferred)

### E-Report & Export

- **REPT-03**: Sistem mengekstrak data mentah ringkasan penjualan harian ke dalam format file CSV (Excel) — *Actually shipped in v1.0 Phase 3 Plan 04, but originally deferred. Requirement met.*

### Notifications

- **NOTF-01**: Sistem mengirimkan notifikasi ringkasan performa penjualan mingguan otomatis langsung ke nomor WhatsApp pemilik setiap hari Minggu jam 21.00 WIB — *Deferred to v2*

## Out of Scope (Still Valid)

| Feature | Reason |
|---------|--------|
| Aplikasi pelanggan & menu digital QR Code | Bukan fokus manajerial pemilik; kompleksitas di luar MVP |
| Sistem antrean & layar dapur (KDS) | Kompleksitas operasional dapur di luar cakupan analitik |
| Manajemen inventaris & stok gudang | Ditangguhkan ke versi selanjutnya; memerlukan entitas data tambahan |
| Integrasi gerbang pembayaran (e-wallet/transfer) | Tidak ada transaksi dalam aplikasi; scope independen |
| Manajemen multi-outlet / multi-cabang | v1 didesain khusus single-outlet; struktur DB tidak mengakomodasi multi-tenant |
| Sistem loyalitas & membership | Tidak ada interaksi sistem dengan konsumen |
| Ulasan, rating menu, & feedback pelanggan | Tidak ada interaksi sistem dengan konsumen |
| Real-time POS integration | Memerlukan hardware dan protokol integrasi eksternal; di luar timeline 8 minggu |
