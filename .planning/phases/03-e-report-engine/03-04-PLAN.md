  <action>
    Create a custom CSV generator that produces Excel-compatible output for Indonesian locale.

    - Import `ReportData` from '../types/report'.
    - Export function `generateReportCSV(data: ReportData): void`.
    - Define headers: `['Tanggal', 'Omset (Rp)', 'Menu Terlaris', 'Hari Tercatat']`.
    - Implement `escapeCell(val: string): string`:
      1. Convert any value to string.
      2. **CSV injection mitigation**: If the string starts with `=`, `+`, `-`, or `@`, prefix with a tab character (`\t`) to prevent Excel formula interpretation (RESEARCH.md Security Domain mitigation for formula injection).
      3. If the string contains `;`, `"`, or `\n`, wrap the entire value in double quotes and replace any internal `"` with `""` (RFC 4180 escaping).
      4. Otherwise return the string as-is.
    - Map rows: `data.rows.map(r => [r.date, formatRupiah(r.revenue), r.topMenu, String(r.dayCount)].map(escapeCell).join(';'))`.
    - Build CSV content: `[headers.join(';'), ...rows].join('\r\n')` (Windows-style CRLF for Excel compatibility).
    - Prepend UTF-8 BOM: `const BOM = '\uFEFF'; const blob = new Blob([BOM + csvContent], { type: 'text/csv;charset=utf-8;' })`.
    - Trigger download:
      ```typescript
      const url = URL.createObjectURL(blob);
      const link = document.createElement('a');
      link.href = url;
      const safeName = data.outlet.name.replace(/[^a-zA-Z0-9]/g, '_');
      link.download = `Laporan_${safeName}_${data.period.start}_${data.period.end}.csv`;
      document.body.appendChild(link);
      link.click();
      document.body.removeChild(link);
      URL.revokeObjectURL(url);
      ```
      Per RESEARCH.md Pitfall 4, always call `URL.revokeObjectURL(url)` after the click to prevent memory leaks.
    - Per D-26, columns are Tanggal, Omset (Rp), Menu Terlaris, Hari Tercatat.
    - Per D-29, file naming follows the convention.
    - Per D-36, generation happens entirely client-side.
    - Per RESEARCH.md recommendation, no external CSV library is used — custom generator is ~30 LOC and sufficient.
  </action>
