<objective>
Prepare the frontend and backend for split deployment on Render. The frontend static build must know the backend API URL at build time, CORS must allow the production frontend origin, and both builds must compile cleanly. Produce a render.yaml blueprint.

Purpose: Split deploy (D-61) means the frontend and backend run on separate origins. The frontend cannot rely on Vite dev proxy in production; it needs an absolute API base URL.
Output: Env-aware frontend API client, verified backend+frontend builds, and Render infrastructure blueprint.
</objective>
