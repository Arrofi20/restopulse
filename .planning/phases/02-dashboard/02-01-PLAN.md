<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client→/api/dashboard | Untrusted HTTP requests, must be JWT-authenticated and date-range validated |
| Express→SQLite | Prisma queries with outlet_id scoping — no raw SQL |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-01 | Spoofing | /api/dashboard | high | mitigate | JWT Bearer token verified by authMiddleware; all queries scoped to decoded outletId |
| T-02-02 | Tampering | CORS config | medium | mitigate | Explicit `cors({ origin, credentials })` prevents arbitrary origins; only localhost:5173 in dev |
| T-02-03 | Information Disclosure | /api/dashboard | high | mitigate | Financial data only returned for authenticated outlet; no data leakage across outlets |
| T-02-04 | Denial of Service | /api/dashboard | low | accept | No rate limiting on dashboard (v1 MVP); wide date ranges query pre-computed SalesTrend (O(1) reads) |
| T-02-05 | Elevation of Privilege | authMiddleware | high | mitigate | Existing authMiddleware from Phase 1 enforces Bearer token; token contains userId+outletId |
</threat_model>
