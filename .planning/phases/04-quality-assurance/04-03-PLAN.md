
**Common fixes expected (apply as needed based on findings):**

1. **Touch targets <44px (Major):**
   - If hamburger button (☰, text-2xl) is <44px at 320px: add `min-w-[44px] min-h-[44px]` to the button element, increase padding
   - If sidebar close button (×) is <44px: add `min-w-[44px] min-h-[44px]`, increase tap area
   - If DateFilter preset buttons (px-3 py-1.5 ≈36px tall) are <44px: increase to `py-2.5` (40px + border = 44px) or add `min-h-[44px]`
   - If sidebar nav items (py-2.5 ≈34px): increase to `py-3` (48px)
   - If date inputs are too small: increase padding to `px-3 py-2` and add `min-h-[44px]`

2. **Focus rings missing or invisible:**
   - Verify all interactive elements have `focus:outline-none focus:ring-2 focus:ring-amber-400 focus:border-amber-400` or equivalent

   - If any element lacks a focus indicator, add Tailwind focus classes

3. **Horizontal overflow at 320px:**
   - Check page containers: ensure `max-w-full` or `overflow-x-hidden` on root elements
   - Check tables: ensure `overflow-x-auto` wrapper with `max-w-full`
   - Check chart containers: ensure `max-w-full` and responsive width
   - Check long text strings (outlet name, menu items): add `truncate` or `break-words`

4. **ARIA attributes:**
   - If hamburger missing `aria-expanded` (true/false based on sidebar state): add it
   - If any element has `aria-pressed` but doesn't toggle: fix to reflect actual state
   - If backdrop has incorrect `aria-hidden`: verify it's `true` (screen reader ignores backdrop content)

5. **Color contrast failures:**
   - If any text pair fails ≥4.5:1: adjust the Tailwind color class. For example:
