**Bug triage and fix phase. Per D-55:**
- **Critical:** Fix NOW — blocks sign-off. Examples: feature completely broken, auth bypass, data corruption, security vulnerability
- **Major:** Fix before sign-off. Examples: broken on mobile, accessibility violation (Major), performance below NFR
- **Minor:** Document, defer to v2. Examples: cosmetic issue, nice-to-have improvement

**Process:**

1. **Aggregate all bugs** from the 4 prior plan SUMMARY.md files and accessibility-checklist.md into a single bug list sorted by severity (Critical first, then Major)

2. **For each Critical bug:**
   - Identify the root cause (file + line)
   - Implement the fix
   - Verify with the relevant test suite:
     - API bugs: `npx vitest run --config vitest.config.ts`
     - Frontend bugs: `cd frontend && npm test`
     - Performance: re-run `k6 run k6/load-test.js`
     - Accessibility: manual re-verify at the affected breakpoint
   - Mark as "Fixed" in the SUMMARY.md or checklist

3. **For each Major bug:**
   - Same process as Critical
   - After all Major bugs are fixed, re-run full test suites

4. **Verification gate:** After all Critical+Major fixes:
   - `npx vitest run --config vitest.config.ts` — all API tests pass
   - `cd frontend && npm test` — all frontend tests pass
   - `npx tsc --noEmit` (root) and `cd frontend && npx tsc -b` (frontend) — no type errors
   - Re-run any failing k6 thresholds if performance bugs were fixed
   - Re-run Lighthouse if bundle size or performance bugs were fixed

5. **Update UAT documents** — mark bugs as Fixed in the uat-checklist.md bug summary table

**Important constraint:** Bug fixes must not introduce regressions. Always run the existing test suites before and after each fix. If a fix touches shared code (e.g., format.ts, authMiddleware.ts), run BOTH backend and frontend tests.

**the agent's Discretion on fix approach:**
- Follow existing code patterns (don't refactor or restructure)
- Keep fixes minimal and targeted
- If a bug requires a design decision or architectural change, escalate to a checkpoint:decision
