**General rules for all test files (per D-37 black-box, D-40 in-memory isolation):**
- Each test file imports `createTestDb` + `seedTestData` from `./setup` and `app` from `../app`
- Each top-level `describe` block creates a fresh PrismaClient in `beforeAll`, seeds data, and disconnects in `afterAll`
- All authenticated requests use `authRequest(owner.userId, owner.outletId)` from `./helpers`
- All assertions verify HTTP status + response body shape `{ success, data? }` or `{ success: false, error: { code, message } }`
- Never import service/repository/controller classes directly (black-box per D-37)

Test cases to implement:

1. `POST /api/auth/register` — creates new owner + outlet, returns 201 + JWT:
   - Send `{ username: 'newowner', password: 'securepass123' }`
   - Assert `res.status === 201`, `res.body.success === true`, `res.body.data.token` is a non-empty string, `res.body.data.owner.username === 'newowner'`

   - First register 'dupuser' (should succeed), then register 'dupuser' again
   - Assert `res.status !== 201` (400 or 409), `res.body.success === false`

3. `POST /api/auth/login` — returns JWT for valid credentials:
   - Seed test data (creates outlet + 'testuser'/'testpass123')
   - Send `{ username: 'testuser', password: 'testpass123' }`

   - Assert `res.body.data.owner.username === 'testuser'`

4. `POST /api/auth/login` — rejects invalid password with 401:
   - Send `{ username: 'testuser', password: 'wrongpassword' }`
