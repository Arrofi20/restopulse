<objective>
Implement the complete authentication system for RestoPulse: password hashing with bcryptjs, JWT token generation and verification, Express middleware for route protection, and REST API endpoints for register, login, and logout.

Purpose: Financial data is sensitive; no login = no trust. All subsequent endpoints (sales, reports, admin) depend on this auth layer. The auth system must be secure but not burdensome (no forced resets, simple password policy).
Output: AuthService, JWT utilities, auth middleware, AuthController, and auth routes.
</objective>

Create src/middleware/authMiddleware.ts with exact specifications:

- Import { Request, Response, NextFunction } from 'express'
- Import { verifyToken } from '../lib/jwt'
- Export interface AuthenticatedRequest extends Request { user?: { userId: string; outletId: string } }
- Export function authMiddleware(req: AuthenticatedRequest, res: Response, next: NextFunction): void
  - Extract Authorization header: const authHeader = req.headers.authorization
  - If missing or doesn't start with "Bearer ", respond 401 JSON: { success: false, error: { code: "UNAUTHORIZED", message: "No token provided" } }
  - Extract token: const token = authHeader.split(' ')[1]
  - Call verifyToken(token)
  - If valid, attach req.user = { userId: decoded.userId, outletId: decoded.outletId } and call next()
  - If invalid (catch error), respond 401 JSON: { success: false, error: { code: "UNAUTHORIZED", message: "Invalid token" } }

Note: Since this is the first middleware, create the src/middleware/ directory.
