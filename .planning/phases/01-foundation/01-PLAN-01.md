6. model SalesTrend: id (String, uuid, default), date (DateTime), revenue (Decimal, precision 12, scale 2), menu_popularity (Json), outlet_id (String, FK), created_at (DateTime, default now), updated_at (DateTime, updatedAt)
7. model StatusLog: id (String, uuid, default), action (String), entity_type (String), entity_id (String), old_value (Json, nullable), new_value (Json, nullable), actor_id (String), created_at (DateTime, default now)
8. model DailySalesReport: id (String, uuid, default), period_start (DateTime), period_end (DateTime), total_revenue (Decimal, precision 12, scale 2), transaction_count (Int), top_items (Json), generated_at (DateTime, default now), outlet_id (String, FK)

Add the following indexes:
- @@index([outlet_id, date]) on DailySales
- @@index([outlet_id, date]) on SalesTrend
- @@index([entity_type, entity_id]) on StatusLog
- @@index([outlet_id, period_start, period_end]) on DailySalesReport

Add the following unique constraints:
- @@unique([outlet_id, date]) on DailySales

Add relations:
- OwnerAccount.outlet → Outlet relation (fields: [outlet_id], references: [id])
- DailySales.outlet → Outlet relation (fields: [outlet_id], references: [id])
- SalesTrend.outlet → Outlet relation (fields: [outlet_id], references: [id])
- DailySalesReport.outlet → Outlet relation (fields: [outlet_id], references: [id])

Create .env file in project root with DATABASE_URL="file:./prisma/dev.db" and JWT_SECRET="change-me-in-production" if .env does not exist.
